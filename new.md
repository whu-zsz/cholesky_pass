# Pass 层突破方案文献调研

## 一、方案一：ParSy 的"检查器-执行器"模型

### 核心思想

ParSy 是一个针对稀疏矩阵计算（Cholesky 分解）的并行优化框架，核心解决负载不均衡和同步开销大的问题。

### 工作机制

| 阶段 | 功能 |
|:---|:---|
| **检查阶段 (Inspection)** | 运行时通过"检查器"分析具体数据（矩阵非零元结构），构建任务依赖图（DAG） |
| **任务粗化 (Task Coarsening)** | 引入 LBC 算法，将细粒度任务合并为负载均衡的粗粒度任务集（H-Level set） |
| **执行阶段 (Execution)** | 执行器根据粗化任务图高效并行执行 |

### 量化收益

- ParSy 在多核处理器上的 Cholesky 分解速度是 **MKL Pardiso 的 2.8 倍**
- 是 **PaStiX 的 3.1 倍**

### 落地方式

在 LLVM Pass 中生成"检查器"代码：
1. 运行时读取 `n` 和 `B` 后构建 DAG
2. 生成最优并行调度策略
3. 由执行器执行

## 二、方案二：LLVM Polly 多面体优化

### 核心思想

Polly 是 LLVM 官方的循环优化基础设施，自动进行循环变换（分块、融合）和 OpenMP 并行化。

### 优势

- 经过充分测试的成熟开源框架
- 避开复杂的底层 Pass 开发
- "一键式"优化可能直接突破瓶颈

### 量化收益

研究对比显示，四线程下静态调度性能提升（**8.7×** 和 **10.6×**）远超 Polly——说明正确调度的潜力巨大，同时也提示 Polly 可能是你的捷径。

### 落地方式

目标不是重写 Polly，而是**利用 Polly**：
1. 让 Cholesky 源码满足 SCoP 条件（循环边界为常量/循环不变量，数组访问为仿射形式）
2. 在编译命令中启用 Polly
3. 如果有效，这是最省力的 Pass 层突破方案

## 三、方案三：PLASMA 静态调度

### 核心思想

PLASMA 是多核架构的稠密线性代数库，其 Cholesky 分解采用**静态调度**——通过轻量级运行时或纯静态调度表达丰富并行性。

### 工作机制

- 任务在编译时/初始化时静态地、确定性地分配到各线程
- 无需运行时动态调度，避免竞态和死锁
- 轻量级同步机制（如 progress table + busy-wait）

### 落地方式

研究 PLASMA 源码，学习其静态任务分配策略：
1. 在 Pass 中为每个 tile 的 `cholesky/trsm/madd` 操作分配核心
2. 生成确定性调度代码
3. 使用简单同步机制（避免条件变量）

---

## 四、方案四：自动任务并行化 Pass

### 核心思想

LLVM Pass 自动分析循环依赖，将无依赖循环转换为任务，提交给运行时库（OpenMP/Cilk）并行执行。

### 量化收益

研究实现了类似 Pass，在 **64 线程上取得最高 61 倍加速**。

### 落地方式

让 Pass 识别 Cholesky 中可并行的波前循环，生成调用 OpenMP 任务机制的代码。

**注意**：工程难度较高，但收益巨大。

---

## 五、方案五：Inspector/Executor 处理不规则依赖

### 核心思想

用于处理存在不规则数据依赖的问题（如稀疏 Cholesky）。

### 工作机制

1. **运行时 Inspector**：分析间接数组访问模式，动态构建依赖图
2. **Executor**：根据依赖图调度任务

### 落地方式

虽然处理的是稠密矩阵，但分块带来的块间依赖图是动态的（取决于 `n` 和 `B`）：
1. 程序初始化阶段（读取 `n`、`B` 后）运行 Inspector
2. 构建本次运行的块级依赖图
3. 按图并行执行

## 六、方案对比与建议

| 方案 | 实现难度 | 预期收益 | 推荐优先级 |
|:---|:---|:---|:---|
| Polly 多面体优化 | 极低 | 未知（可能显著） | ⭐⭐⭐⭐⭐（立即尝试） |
| ParSy Inspector/Executor | 中 | 2× ~ 5× | ⭐⭐⭐⭐ |
| PLASMA 静态调度 | 中 | 2× ~ 4× | ⭐⭐⭐⭐ |
| 自动任务并行 Pass | 高 | 10×+ | ⭐⭐⭐ |
| Inspector/Executor | 中高 | 3× ~ 6× | ⭐⭐⭐ |

### 优先行动项

1. **立即尝试 Polly**：在编译命令中加入 `-mllvm -polly -mllvm -polly-parallel`，看看能否自动优化
2. **重点研究 ParSy/PLASMA**：它们的"检查器-执行器"和"静态调度"思想与你的困境完全匹配
3. **长期攻关**：如果前两者效果不理想且时间充裕，开发自动任务并行化 Pass
